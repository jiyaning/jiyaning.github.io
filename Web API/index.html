                                                              # 14.DOM事件-冒泡与捕获

                                                              事件执行3个阶段：捕获、到达目标、冒泡

                                                              事件捕获：从根部节点（document对象）一层一层向文档内找，直到找到事件触发的那个节点，这个过程叫事件捕获；

                                                              到达目标：找到事件触发的那个节点

                                                              冒泡：从事件触发的那个节点，一层一层向文档外找，直到根节点（document对象）的过程叫事件冒泡；

                                                              //事件默认是在冒泡阶段执行；在冒泡阶段，发现父级节点也注册和用户触发行为一样的事件，这些事件函数被执行；
//父亲注册了和用户行为一样的事件，用户触发子元素，父亲也会跟着被触发
//如果孙子没有点击事件,但是用户对孙子有点击行为,但是爷爷注册有点击事件，爷爷同样也会触发点击事件。

// 为什么事件默认是在冒泡阶段执行？用户体验好；让用户知道自己点的地方是正常的想法；
// 第3个参数默认不写或是false的时候，事件默认在冒泡阶段执行，第3个参数是true，事件是在捕获阶段执行
// JS：控制执行过程；（了解） 已经在捕获阶段执行；
var box_3 = document.querySelector(".box_3");
box_3.addEventListener("click", function() {
    alert(3);
}, true);

                                                              # 16. DOM-事件委托

                                                              不是给子元素注册事件,而是给父级注册事件 ，事件默认在冒泡执行

                                                              把事件注册给父级上。点击子元素，冒泡阶段，信号传递到父级的时候，触发父级注册的函数；

                                                              把事件注册在父级的元素身上，利用事件冒泡执行，当事件传播到已经注册了事件的父级元素身上,判断触发事件的DOM （e.target）节点是否是指定的元素，e.target---->DOM节点；

                                                              父级的事件被后代元素代理.

                                                              e.target.nodeName=="LI"       //返回大写的的标签名

e.target.className == "son"   //返回类名

                                                              3

ul.onclick = function(e) {
    // e.target:点击是谁就是谁，LI   DOM节点
    if (e.target.className == "son") {
        alert(1)
    }

                                                              # 25.3. BOM-Web 存储 API

                                                              Web 存储 API 提供了 sessionStorage （会话存储） 和 localStorage（本地存储）两个存储对象来对网页的数据进行添加、删除、修改、查询操作。

                                                              localStorage（本地存储）

                                                              localStorage 在浏览器中存储 key/value 对，拓展了cookie的4K限制，用于长久保存整个网站的数据，保存的数据没有过期时间，直到手动去除。

                                                              //存入数据：localStorage.setItem（）
// 参数1：存入本地哪个字段内。什么名字；键；位置；
// 参数2：存入的数据
// 真实存入（本地浏览器）数据：
localStorage.setItem("a", "dizhi");

// 获取数据：localStorage.getItem（）
// 参数1：键，位置；
// 返回：读取的数据；如果读取一个不存在的数据，返回null;
var info = localStorage.getItem("a");

// 数据覆盖：
localStorage.setItem("a", "9999");

// 清除某条数据：localStorage.removeItem（）
// 参数1：键，位置；
localStorage.removeItem("b");

// 全部清除：localStorage.clear()
localStorage.clear();
